社内の環境で困っていることがあるので相談です。

踏み台サーバ（Windows）に RDP で接続し、そこから SFTP で SI サーバにアクセスして
ビルド済みメディアをダウンロードしています。しかし踏み台のローカルディスク容量が
ほぼゼロのため、ローカルに保存できません。

現在は RDP の「ローカルディスク共有」を使って、踏み台から WinSCP で直接
フロントマシンのディスクに書き込んでいますが、RDP セッションが切れると
転送が中断されてしまい、何度も resume する必要があり手間がかかっています。

踏み台からは SI サーバ以外にも複数のサーバへ SFTP 接続できます。
そこで、踏み台のローカルディスクを使わずに、

「SFTP サーバ A（例: 1.1.1.1）から読み取ったデータを、
　踏み台上でストリーミングしながら、
　SFTP サーバ B（例: 2.2.2.2）へ直接書き込む」

という SFTP→SFTP の中継コピーをしたいです。

WinSCP の .NET API（PowerShell）には Stream を扱う API があると聞きましたが、
この方法で「踏み台のディスクを使わずに SFTP→SFTP のストリーミング転送」を
実現することは可能でしょうか？

可能であれば、実現方法やサンプルコードのイメージを教えてください。
